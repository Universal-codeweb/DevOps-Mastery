# Day 15 – TypeScript Deeper Concepts

This session focused on advanced TypeScript features that improve code flexibility, reusability, and type safety. The practice included **Union Types**, **Optional Types**, **Generics**, and the **Partial Utility Type** by implementing them with variables, objects, interfaces, classes, functions, and arrays.

---

## Topics Covered

* Union Types (`|`)
* Union Types with Objects
* Union Types in Function Parameters
* Arrays with Union Types
* Optional Properties (`?`)
* Optional Constructor Parameters
* Generic Functions (`<T>`)
* Generic Arrow Functions
* Generic Arrays
* Utility Type – `Partial<T>`

---

## Code Explanation

### Union Types

Implemented union types to allow variables and object properties to store more than one data type.

```ts
let age: number | string;
```

The same concept was extended to custom object types, allowing a variable to hold different object structures using union types.

```ts
type universe | omniverse
```

This provides flexibility while maintaining strict type checking.

---

### Union Types in Functions

Created a function that accepts multiple data types as parameters.

```ts
function tellUniverse(data: string | boolean)
```

The function accepts either a string or a boolean value, demonstrating how union types can be used in function parameters.

---

### Union Type Arrays

Created arrays capable of storing multiple allowed data types.

```ts
let value: (string | number)[]
```

This allows the same array to contain both numbers and strings while preventing unsupported data types.

---

### Optional Properties

Used optional properties inside an interface.

```ts
interface room {
    isClean?: boolean;
}
```

Objects can be created with or without the optional property, making interfaces more flexible.

---

### Optional Constructor Parameters

Created a class with an optional constructor parameter.

```ts
class Car {
    model?: string;
}
```

Objects can be initialized using only the required information or include additional optional details when available.

---

### Optional Properties with Type Aliases

Applied optional properties using a type alias.

```ts
type series {
    year?: number;
}
```

Some objects include the optional property while others omit it, demonstrating flexible object creation.

---

### Utility Type – `Partial<T>`

Implemented TypeScript's built-in `Partial` utility type.

```ts
Partial<User>
```

`Partial<T>` converts every property of a type into an optional property, allowing objects to be created without providing all fields.

---

### Generic Functions

Created reusable generic functions using type parameters.

```ts
function speakToUniverse<T>(value: T): T
```

The function accepts any data type and returns the same type, preserving its original type information.

Another generic function was created to print values without returning them.

```ts
function notSpeakToUniverse<T>(value: T): void
```

---

### Generic Arrow Function

Implemented a reusable generic arrow function.

```ts
let displayResult = <T>(data_item: T[]): T[]
```

The function works with arrays of any data type and returns a new array while preserving the original type.

---

### Generic Arrays

Used the generic function to create arrays of different types.

```ts
displayResult<number>()
displayResult<string>()
```

The returned arrays maintain their respective types, ensuring that only valid values can be added later.
