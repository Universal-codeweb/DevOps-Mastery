# Day 22 – React Lists & Static Rendering

## React Lists

Today I started learning about **Lists** in React. Before rendering data dynamically, I understood how to store multiple values inside an array and display them manually using list elements.

This helped me understand the difference between **storing data** and **rendering data**. Although the array was created, I manually displayed each student name to prepare for learning React's `.map()` function in the next step.

---

# Code Implemented

```tsx
function StudentList() {

    const students = [
        "Toko",
        "Universe",
        "Yuzhou",
        "Uno"
    ];

    return (
        <>
            <h1>Students</h1>

            <ul>
                <li>Toko</li>
                <li>Universe</li>
                <li>Yuzhou</li>
                <li>Uno</li>
            </ul>
        </>
    );
}

export default StudentList;
```

---

# Code Explanation

## Creating an Array

```tsx
const students = [
    "Toko",
    "Universe",
    "Yuzhou",
    "Uno"
];
```

An array is created to store multiple student names in a single variable.

Instead of creating separate variables for every student, arrays allow related data to be grouped together, making it easier to manage and display later.

---

## Rendering the Heading

```tsx
<h1>Students</h1>
```

Displays the heading of the page.

---

## Creating an Unordered List

```tsx
<ul>
```

The `<ul>` tag creates an unordered list that holds multiple list items.

---

## Rendering List Items

```tsx
<li>Toko</li>
<li>Universe</li>
<li>Yuzhou</li>
<li>Uno</li>
```

Each `<li>` represents one student.

Even though the student names are already stored inside an array, they are written manually here. This is called **static rendering** because every list item is explicitly written in JSX.

This implementation helps understand why React later uses `.map()` to generate list items automatically.

---

## Exporting the Component

```tsx
export default StudentList;
```

Exports the component so it can be imported and rendered inside `App.tsx`.

---

# Static Rendering

In this implementation, every student is written manually.

```tsx
<li>Toko</li>
<li>Universe</li>
<li>Yuzhou</li>
<li>Uno</li>
```

If another student needs to be added, the JSX must also be modified manually.

---

# Why Create an Array First?

Although the array is not used yet, creating it is the first step toward **dynamic rendering**.

Later, React will use the array with the `.map()` function to automatically create list items, avoiding repeated code and making the application easier to maintain.
