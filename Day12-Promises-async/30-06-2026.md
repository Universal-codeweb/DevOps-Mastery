## 1. Creating a Promise

Each function returns a Promise.

```javascript
return new Promise((resolve, reject) => {

});
```

- `new Promise()` creates a new Promise object.
- `resolve()` is called when the task completes successfully.
- `reject()` is called when the task fails.
- The Promise remains **Pending** until either `resolve()` or `reject()` is executed.

---

## 2. Simulating an Asynchronous Task

```javascript
setTimeout(() => {

}, 1500);
```

- `setTimeout()` delays the execution of code.
- It is used here to simulate asynchronous work such as downloading data, reading files, or calling an API.
- The delay values are:
  - `1500 ms` → Walk Dog
  - `6500 ms` → Read Book
  - `10500 ms` → Learn Astronomy

---

## 3. Success and Failure Handling

```javascript
const dogWalked = true;

if (dogWalked) {
    resolve("You walked the dog.");
} else {
    reject("You didn't walk the dog.");
}
```

- A boolean variable determines whether the task succeeds or fails.
- If the value is `true`, `resolve()` is executed.
- If the value is `false`, `reject()` is executed.
- Only one of them can execute for a Promise.

The same logic is used in all three functions.

---

## 4. Promise Chaining

```javascript
walkDog()
    .then(...)
    .then(...)
    .then(...)
```

- The first Promise starts by calling `walkDog()`.
- After it resolves, the next `.then()` executes.
- Each `.then()` returns another Promise, creating a sequence of asynchronous tasks.

Execution order:

```
Walk Dog
    ↓
Read Book
    ↓
Learn Astronomy
```

---

## 5. Using `.then()`

```javascript
.then((value) => {
    console.log(value);
    return readBook();
})
```

- `.then()` executes only when the previous Promise is fulfilled.
- `value` contains the data passed by `resolve()`.
- `console.log()` displays the resolved message.
- Returning `readBook()` starts the next Promise only after the previous one completes.

---

## 6. Returning the Next Promise

```javascript
return readBook();
```

Returning the Promise is important because it tells JavaScript to wait for `readBook()` to complete before moving to the next `.then()`.

Without `return`, the Promise chain would not wait correctly.

---

## 7. Error Handling with `.catch()`

```javascript
.catch((error) => {
    console.log(error);
});
```

- `.catch()` handles any rejected Promise in the chain.
- If one Promise fails, the remaining `.then()` blocks are skipped.
- The error passed to `reject()` is received by the `error` parameter.

Example:

```javascript
const learnedAstro = false;
```

Output:

```text
You walked the dog.
You read the book.
You didn't learn astronomy.
```

---
