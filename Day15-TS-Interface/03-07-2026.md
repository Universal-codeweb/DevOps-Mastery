# 1. Object Types

An object type defines the structure of an object directly while declaring it. Every property must match the specified type.

### Syntax

```ts
const object: {
    property: type;
} = {
    property: value
};
```

### Practiced

Created objects for:

* **Car**

  * `name`
  * `year`
  * `hasLicense`

* **Employee**

  * `name`
  * `age`
  * `DOJ` (Date of Joining)
  * `Exp` (Experience)

* **Book**

  * `name`
  * `year`
  * `author`
  * `publisher`

These objects helped understand how TypeScript validates object properties and prevents assigning incorrect data types.

---

# 2. Interfaces

An interface is a reusable blueprint that defines the structure of an object. Instead of writing object types repeatedly, an interface can be created once and reused throughout the project.

### Syntax

```ts
interface InterfaceName {
    property: type;
}
```

### Practiced

Created a `Movie` interface containing:

* `name`
* `genre`
* `year`
* `director`

Then created a `Movie` object using that interface.

This demonstrated how interfaces improve code readability, reusability, and maintainability.

---

# 3. Functions with Interfaces

Interfaces can also be used as parameter types for functions.

### Syntax

```ts
function functionName(parameter: InterfaceName): void {

}
```

### Practiced

Created a function:

```ts
function mname(movie: Movie): void
```

The function accepts only objects that satisfy the `Movie` interface and prints the movie name.

The function was called in two ways:

* Passing an object directly.
* Understanding that existing objects implementing the interface can also be passed.

---

# 4. Function Return Types

TypeScript allows specifying what a function should return.

Common return types include:

* `void`
* `number`
* `string`
* `boolean`
* Custom interfaces

### Practiced

Used:

```ts
:void
```

for functions that only perform an action without returning a value.

Used:

```ts
:number
```

for methods that return numerical values.

---

# 5. Interfaces with Classes

Interfaces are not limited to objects—they can also define contracts for classes.

A class implementing an interface must provide implementations for all methods declared in that interface.

### Syntax

```ts
interface Calculator {

}

class Cal implements Calculator {

}
```

### Practiced

Created a `Calculator` interface with methods:

* `multiply()`
* `divide()`

Then implemented those methods inside the `Cal` class and created an object to call them.

This demonstrated how interfaces enforce consistency across class implementations.

> **Note:** In your current implementation, `multiply()` returns `a + b`. For multiplication, it should return `a * b`.

Defined a reusable structure for movie objects.


## Calculator Interface

Defined a contract containing two mathematical operations.

```ts
interface Calculator {
    multiply(a: number, b: number): number;
    divide(a: number, b: number): number;
}
```

The interface specifies both the parameters and return types of each method.
