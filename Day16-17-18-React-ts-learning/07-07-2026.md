# Day 16 – React + TypeScript Setup & Components

## React + Vite + TypeScript

A React project was created using **Vite** with **TypeScript**. Unlike traditional HTML projects, React applications start from `main.tsx`, where React takes control of the application and renders it inside the root element.

### Project Structure

```text
src/
│
├── main.tsx
├── App.tsx
├── Greeting.tsx
├── App.css
└── index.css
```

---

## React Rendering Flow

React renders the application in the following order:

```text
index.html
      ↓
<div id="root"></div>
      ↓
main.tsx
      ↓
createRoot()
      ↓
render(<App />)
      ↓
App Component
      ↓
Child Components
      ↓
Browser UI
```

* `index.html` provides the root container.
* `createRoot()` creates a React Root to manage that container.
* `render()` displays the React component tree inside the root.

## JSX

JSX (JavaScript XML) is a syntax that allows writing HTML-like code inside TypeScript or JavaScript.

```tsx
<h1>Welcome to React</h1>
```

JSX is **not HTML**. During development, Vite transforms JSX into JavaScript that React understands.

---

## JavaScript Inside JSX

Curly braces `{}` allow JavaScript expressions to be evaluated inside JSX.

```tsx
const name = "Yuzhou";

<h1>Hello {name}</h1>
```

Output:

```text
Hello Yuzhou
```

Expressions such as variables, calculations, method calls, and function calls can be placed inside `{}`.

```tsx
<p>{10 + 20}</p>
```

Output:

```text
30
```

## Components

A React component is a JavaScript or TypeScript function that returns JSX.

```tsx
function Greeting() {
    return <h1>Hello</h1>;
}
```

React executes the component function and renders the returned JSX.

```text
<Greeting />

↓

Greeting()

↓

Returns JSX

↓

Rendered on the screen
```

Component names must begin with an uppercase letter.

```tsx
<Greeting />   // Component
<button />     // HTML element
```

---

## Reusable Components

The same component can be used multiple times.

```tsx
<Greeting />
<Greeting />
<Greeting />
```

Although only one function exists, React executes it once for each component usage, creating multiple UI elements.

---

## Props

Props are inputs passed from a parent component to a child component, similar to function arguments in JavaScript.

```tsx
type GreetingProps = {
    name: string;
}

function Greetings(props: GreetingProps) {
    return (
        <h1>Hello {props.name} 🥱</h1>
    );
}

export default Greetings;
```

Usage:

```tsx
<Greetings name="Toko" />
<Greetings name="Universe" />
<Greetings name="React" />
```

Each component receives different data through `props`.

Execution flow:

```text
<Greetings name="Toko" />

↓

Greetings({
    name: "Toko"
})

↓

props.name

↓

"Toko"

↓

<h1>Hello Toko 🥱</h1>
```

---

## TypeScript Props

A custom type defines the structure of props.

```tsx
type GreetingProps = {
    name: string;
}
```

This tells TypeScript that the component requires a `name` property of type `string`.

If the required prop is missing or has the wrong type, TypeScript reports a compile-time error before the application runs.

---

## Component Composition

Components can be combined to build larger interfaces.

```tsx
<App />
    │
    ├── <Greetings name="Toko" />
    ├── <Greetings name="Universe" />
    └── <Greetings name="React" />
```

Each child component executes independently and returns its own JSX, allowing the UI to be built from small, reusable pieces.
