# Day 20 – Introduction to `useEffect`

## React `useEffect`

Today I started learning the `useEffect` Hook in React. Before understanding the syntax, I learned why React introduced `useEffect` and what problem it solves.

A React component is just a JavaScript function. Whenever the state changes, React executes the component again (re-render). Any code written directly inside the component body will execute every time the component renders.

Example:

```tsx
function UseEffectLearning() {

    const [count, setCount] = useState(0);

    console.log("Component Rendered");

    return (
        <>
            <h1>{count}</h1>
            <button onClick={() => setCount(count + 1)}>
                Increase
            </button>
        </>
    );
}
```

Every click on the button causes the component to execute again, therefore the console message is printed after every render.

---

## Why `useEffect`?

Some operations should not be performed while React is rendering the UI.

Examples include:

- Fetching data from an API
- Updating the browser title
- Accessing Local Storage
- Starting timers
- Adding event listeners

These operations are called **Side Effects** because they perform work outside of rendering the user interface.

React provides the `useEffect` Hook to execute this type of work **after the component has rendered**.

---

## Code Implemented

```tsx
import { useEffect, useState } from "react";

function UseEffectLearning() {

    const [count, setCount] = useState(0);

    useEffect(() => {
        console.log("Effect Executed");
    });

    return (
        <>
            <h1>{count}</h1>

            <button onClick={() => setCount(count + 1)}>
                Increase
            </button>
        </>
    );
}

export default UseEffectLearning;
```

---

## Code Explanation

### Importing Hooks

```tsx
import { useEffect, useState } from "react";
```

- `useState` is used to create and manage state.
- `useEffect` is used to perform side effects after rendering.

---

### Creating State

```tsx
const [count, setCount] = useState(0);
```

- `count` stores the current value.
- `setCount` updates the state.
- `0` is the initial value during the first render.

Whenever `setCount()` is called, React stores the new value and re-renders the component.

---

### Creating an Effect

```tsx
useEffect(() => {
    console.log("Effect Executed");
});
```

This effect has **no dependency array**, so React executes it **after every render**.

Execution order:

```
Component Renders
        ↓
UI is displayed
        ↓
useEffect executes
```

If the component renders again because of a state update, the effect runs again.

---

### Rendering the UI

```tsx
<h1>{count}</h1>
```

Displays the current state value.

Whenever the state changes, React creates a new render and updates the displayed value.

---

### Updating State

```tsx
<button onClick={() => setCount(count + 1)}>
    Increase
</button>
```

When the button is clicked:

1. `setCount()` receives the new value.
2. React stores the updated state.
3. The component renders again.
4. The UI displays the updated count.
5. `useEffect()` executes after the render is completed.

---

## Render Flow

```
Component Loads
        ↓
useState() returns initial value
        ↓
Component returns JSX
        ↓
Browser displays UI
        ↓
useEffect executes
```

When the button is clicked:

```
Button Click
        ↓
setCount(count + 1)
        ↓
React stores new state
        ↓
Component re-renders
        ↓
Browser updates UI
        ↓
useEffect executes again
```

---

## What I Understood

- React components execute again whenever their state changes.
- Code written directly inside the component executes during every render.
- `useEffect` is used to perform side effects after React finishes rendering.
- Without a dependency array, `useEffect` runs after every render.
- Rendering the UI and performing side effects are two separate responsibilities in React.
