# 🌐 What is NestJS?

NestJS is a backend framework built on top of **Node.js** and **Express** (or Fastify). It uses **TypeScript** by default and follows a modular architecture, making applications easier to organize, maintain, and scale.

### Learned

- What is NestJS
- Why NestJS is used
- Difference between Express and NestJS
- NestJS Architecture
- Role of Modules, Controllers, and Services

```text
Response
```

# 📄 main.ts

Learned that `main.ts` is the entry point of the application. It creates the NestJS application and starts the server.

```typescript
async function bootstrap() {
  const app = await NestFactory.create(AppModule);
  await app.listen(3000);
}
bootstrap();
```

---

# 📦 Modules

Learned that a **Module** is used to organize related components into a single unit.

Example:

```text
Student Module
│
├── Student Controller
├── Student Service
└── Student DTO
```

Modules help keep large applications clean and modular.

---

# 🎮 Controllers

Learned that a **Controller** receives incoming HTTP requests and sends responses back to the client.

Example:

```typescript
@Controller()
export class AppController {
  @Get()
  getHello() {
    return "Hello World!";
  }
}
```

---

# ⚙️ Services

Learned that a **Service** contains the business logic of the application.

Example:

```typescript
@Injectable()
export class AppService {
  getHello(): string {
    return "Hello World!";
  }
}
```

Instead of writing logic inside the controller, the controller delegates the work to the service.
