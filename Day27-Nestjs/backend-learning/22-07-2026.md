# 🌐 What is an API?

An API (Application Programming Interface) acts as a bridge between the frontend and the backend. Instead of communicating directly with the database, the frontend sends requests to the backend, which processes the request and returns the appropriate response.

### Request Flow

```text
Frontend
    │
    ▼
Controller
    │
    ▼
Service
    │
    ▼
Database (Later)
```

---

# 🌍 Understanding REST API

REST (Representational State Transfer) is a standard way of designing web APIs using HTTP methods.

### Common HTTP Methods

| Method | Purpose | Example |
|---------|----------|----------|
| GET | Retrieve data | Get all students |
| POST | Create data | Add a student |
| PUT | Update existing data | Update student details |
| DELETE | Remove data | Delete a student |

# 🛠️ Creating a Resource

Generated a new **Students** resource using the Nest CLI.

```bash
nest g resource students
```

Selected:

- REST API
- Generate CRUD entry points

NestJS automatically created the following structure:

```text
src/
└── students/
    ├── dto/
    ├── entities/
    ├── students.controller.ts
    ├── students.service.ts
    ├── students.module.ts
    └── students.controller.spec.ts
```

---

# 📂 Understanding the Generated Files

### students.module.ts

Registers and groups all components related to the Students feature.

### students.controller.ts

Handles incoming HTTP requests and forwards them to the service.

### students.service.ts

Contains the business logic and processes the application's data.

### dto/

Stores Data Transfer Objects, which define the expected request data.

### entities/

Represents the structure of the data model (used later with databases).

# 🎯 Understanding Controllers

A Controller acts as the entry point for HTTP requests.

Example:

```typescript
@Get()
findAll() {
  return this.studentsService.findAll();
}
```

### Learned

- `@Get()` handles GET requests.
- The controller receives the request.
- It delegates the work to the service.
- It returns the response to the client.

---

# ⚙️ Understanding Services

Services contain the business logic of the application.

Example:

```typescript
findAll() {
  return [
    {
      id: 1,
      name: "Uno",
      department: "CSE"
    }
  ];
}
```

### Learned

- Services process data.
- Controllers should not contain business logic.
- Separating responsibilities makes applications easier to maintain and scale.

---

# 🏷️ Understanding Decorators

NestJS uses decorators to define the role of classes and methods.

### Class Decorator

```typescript
@Controller('students')
```

Maps the controller to the `/students` route.

### Method Decorator

```typescript
@Get()
```

Specifies that the method handles HTTP GET requests.

# 🔄 Request Lifecycle

When a client requests:

```text
GET /students
```

The flow is:

```text
Browser/Postman
        │
        ▼
StudentsController
        │
        ▼
StudentsService
        │
        ▼
Return Data
        │
        ▼
NestJS converts it to JSON
        │
        ▼
Client receives the response
```
